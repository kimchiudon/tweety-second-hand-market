# Tweety

Tiny보다 더 작고 가벼운 느낌의, 보안을 우선한 중고 거래 웹 플랫폼입니다. 수업 자료의 최소 요구사항 7개(가입, 상품 등록·조회, 사용자 소통, 신고·차단, 사용자 간 송금, 검색, 관리자 관리)를 모두 구현했습니다.

## 주요 기능

- 회원가입·로그인, 중복 불가 닉네임, 소개글 및 비밀번호 변경, 내 판매·구매 내역
- 전체·디지털·패션·생활/가구·도서·스포츠·기타 상품 분류 모아보기와 분류 내 검색
- PNG/JPEG 사진을 상품·채팅에 각각 최대 10장 첨부하고 갤러리로 조회(11장 선택 시 브라우저에서 즉시 안내하고 서버에서도 재검사)
- 상품 상세에서 바로 시작하는 판매자와의 상품별 1:1 채팅, 채팅 사진 첨부
- 10초 간격으로 갱신되는 읽지 않은 채팅 알림 배지와 대화별 읽음 처리
- 별도 송금 확인 화면에서 거래 내용을 검토한 뒤 구매자 포인트를 판매자에게 원자적으로 이전
- 사용자 차단·해제 및 차단 관계의 상품·채팅·구매 제한
- 상품/사용자 신고와 실제 대화 참여자만 접수할 수 있는 채팅 신고, 누적 신고에 따른 자동 숨김·정지
- 관리자 대시보드에서 신고 처리, 차단 관계 관리, 계정 활동 정지·해제, 신고 상품 삭제
- 활동 정지 계정은 로그인·내역·기존 채팅 조회만 허용하고 판매·구매·메시지 전송은 서버에서 제한
- 현재 비밀번호와 확인 문구를 요구하는 회원탈퇴, 개인정보 익명화·전 기기 세션 폐기·판매 상품 즉시 숨김
- 구매 시 구매자 포인트를 판매자에게 원자적으로 송금하고 마이페이지에서 내역 확인
- 반응형 웹 UI와 키보드/스크린리더를 고려한 기본 접근성

## 환경 설정

- Python 3.11 이상
- Python 패키지 설치 도구(`pip`)와 이미지 안전 처리용 Pillow
- Linux, macOS, WSL에서 실행 가능

```bash
git clone https://github.com/kimchiudon/tweety-second-hand-market.git
cd tiny-second-hand-shopping-platform
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e .
python -m unittest discover -v
python -m tiny_market
```

브라우저에서 `http://127.0.0.1:8000`에 접속합니다. 최초 실행 시 프로젝트 루트에 `market.db`가 자동 생성됩니다.

### 관리자 계정 생성

서버를 중지한 상태에서 다음 명령을 실행하고, 화면에 표시되지 않는 비밀번호 입력란에 안전한 비밀번호를 입력합니다.

```bash
python -m scripts.create_admin
```

관리자로 로그인하면 상단의 `관리` 메뉴가 나타납니다.

### 선택 환경 변수

| 이름 | 기본값 | 설명 |
|---|---|---|
| `TINY_MARKET_ENV` | `development` | 공개 배포 시 `production`으로 설정해 Secure 쿠키와 HSTS 활성화 |
| `TINY_MARKET_DB` | `./market.db` | SQLite 데이터베이스 절대/상대 경로 |
| `TINY_MARKET_HOST` | `127.0.0.1` | 바인딩 주소 |
| `TINY_MARKET_PORT` | `8000` | 서버 포트(1~65535) |

> 포함된 `wsgiref` 서버는 수업·로컬 검증용입니다. 인터넷에 직접 공개하지 말고, 운영 시 HTTPS 리버스 프록시와 다중 프로세스 WSGI 서버 뒤에서 실행해야 합니다.

## 테스트

```bash
python -m unittest discover -v
```

테스트 30개는 비밀번호 해싱과 로그인 속도 제한, CSRF 차단, SQL 삽입·XSS 방어, 상품명 한 글자 이상 일치 검색, 카테고리 허용 목록과 검색·차단·정지 판매자 조건 결합, 소유권 검사, 닉네임 가입·중복 방지, 회원탈퇴의 비밀번호 재확인·익명화·전 세션 폐기·상품 숨김·재로그인 및 관리자 복구 차단, 상품·채팅 최대 10장 이미지, WebP·위장 파일 차단, 채팅 사진 접근 통제, 채팅 알림·읽음 처리, 실제 대화 참여자만 가능한 채팅 신고와 관리자 확인, 사용자 차단 강제, 정지 계정의 로그인·조회 허용과 거래·메시지 제한, 관리자 신고·차단·계정 정지 관리, 신고상품 삭제 권한, 상품별 1:1 메시지 격리, 송금 확인 화면, 잔액 부족, 중복 구매 방지, 신고 누적 자동 차단, 보안 헤더, 무세션 상태 확인, 관리자 최초 생성의 멱등성을 검증합니다.

## Render 배포

저장소 루트의 `render.yaml`은 Singapore 리전의 Python 웹 서비스, HTTPS, 운영용 Gunicorn, 상태 확인 경로와 5GB 영구 디스크를 선언합니다. Render Dashboard에서 이 저장소를 Blueprint로 연결하고 `TINY_MARKET_ADMIN_PASSWORD`를 비밀값으로 입력하면 최초 관리자 계정이 한 번만 생성됩니다.

> SQLite DB와 사진을 유지하려면 유료 웹 서비스와 영구 디스크가 필요합니다. 무료 인스턴스로 설정을 바꾸면 재배포·재시작 때 회원, 채팅, 거래, 사진이 사라질 수 있습니다.

## 보안 설계 요약

- 비밀번호: 사용자별 128-bit salt + PBKDF2-HMAC-SHA256 600,000회
- 세션: 256-bit 난수 토큰, DB에는 SHA-256 해시만 보관, 로그인 시 회전, 8시간 만료
- 요청 위조: 모든 상태 변경 POST 요청에 세션 기반 CSRF 토큰 적용
- 입력/출력: 길이·형식·허용 목록 검증, 검색값을 포함한 SQL 매개변수 바인딩, HTML 문맥 이스케이프
- 관리자 권한: 모든 `/admin` 경로에서 서버 DB의 세션 토큰 해시·만료·사용자 ID·현재 관리자 역할을 재검사하며 요청의 `mb_level`·`role` 값은 신뢰하지 않음
- 사진: PNG/JPEG만 허용하고 확장자·MIME·시그니처 일치와 전체 디코딩을 검증한 뒤 최대 4096px로 재인코딩해 메타데이터 제거(파일당 8MB, 요청당 32MB, 최대 10장, 원본 최대 5천만 픽셀)
- 채팅 사진: 추측하기 어려운 난수 파일명에 더해 대화 참가자만 인증된 전용 경로로 조회 가능
- 접근 통제: 객체마다 판매자/구매자/관리자 권한을 서버에서 재검사
- 관리자도 사용자의 비밀번호를 조회하거나 임의 변경할 수 없음. 복구 기능 추가 시 일회용·만료형 사용자 주도 재설정 방식 권장
- 거래: `BEGIN IMMEDIATE` 트랜잭션과 조건부 UPDATE로 이중 구매·잔액 마이너스 방지
- 브라우저: CSP, clickjacking 차단, MIME sniffing 차단, Referrer/Permissions Policy
- 악용 대응: 계정별·IP 전체 로그인 속도 제한, 존재하지 않는 계정도 동일 비용으로 비밀번호 검증, 중복 신고 금지, 감사 로그(IP 원문 대신 해시 저장)

### 기존 업로드 마이그레이션

이전 버전에서 만든 DB에 WebP 또는 재인코딩 전 파일이 있다면 서버를 중지한 뒤 한 번만 실행합니다. 참조 중인 사진을 PNG/JPEG로 안전하게 다시 저장하고 DB 파일명을 함께 갱신합니다.

```bash
python -m scripts.sanitize_uploads
```

## 알려진 제한

- 포인트는 교육용 내부 원장으로 실제 금융 결제나 현금 송금이 아닙니다.
- `Tweety` 마스코트는 이 교육 프로젝트를 위해 만든 오리지널 노란 오리 디자인이며, 동명의 기존 캐릭터·권리자와 관련이 없습니다.
- 상품별 1:1 채팅 알림은 10초 간격으로 갱신되며 종단간 암호화가 아닙니다.
- 예제는 단일 SQLite 인스턴스를 전제로 합니다. 대규모 운영에는 외부 DB, 중앙 속도 제한, 모니터링이 필요합니다.

## AI 활용

Codex를 요구사항 정리, 위협 모델링, 구현, 테스트 케이스 생성, 브라우저 검수, 문서·PDF 초안 제작에 활용했습니다. 생성된 결과는 자동 테스트와 수동 브라우저 점검으로 검증했으며, 최종 보안 판단과 제출 책임은 작성자에게 있습니다.

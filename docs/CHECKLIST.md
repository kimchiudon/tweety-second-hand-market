# 구현·보안 테스트 체크리스트

기준일: 2026-07-21

## 기능 요구사항

- [x] 사용자가 중복되지 않는 아이디와 닉네임으로 가입하고 로그인할 수 있다.
- [x] 소개글과 비밀번호를 변경하고 내 판매·구매 내역을 조회할 수 있다.
- [x] 누구나 노출 중인 상품 목록과 상세를 볼 수 있다.
- [x] 로그인 사용자는 검증·재인코딩된 PNG/JPEG 사진을 최대 10장까지 상품에 등록하고 본인 상품만 수정·삭제할 수 있다.
- [x] 검색어의 글자 하나 이상이 상품명에 포함되면 결과에 표시하며 특수문자 입력도 안전하게 처리한다.
- [x] 상품 상세에서 판매자와 상품별 1:1 채팅을 시작할 수 있고 전체 채팅은 제공하지 않는다.
- [x] 채팅 한 메시지에 사진을 최대 10장까지 안전하게 첨부하고 새 메시지 알림과 읽음 처리를 제공한다.
- [x] 전용 확인 화면에서 상품·판매자·송금액·잔액을 확인한 뒤 구매자 포인트가 판매자에게 이전된다.
- [x] 상품·사용자를 신고하고 중복 신고를 막는다.
- [x] 사용자가 다른 사용자를 차단·해제하고 차단 관계에서는 상품·채팅·구매를 이용할 수 없다.
- [x] 상품 신고 3회 시 자동 숨김, 사용자 신고 5회 시 일반 계정 자동 정지된다.
- [x] 관리자가 사용자 신고를 처리하고 차단 관계를 해제하며 계정 활동을 정지·복구하고 신고 상품을 삭제할 수 있다.
- [x] 활동 정지 사용자는 로그인해 내역과 기존 채팅을 읽을 수 있지만 상품 등록·수정·구매와 메시지 전송은 할 수 없다.

## 보안 체크리스트

- [x] 비밀번호를 평문으로 저장·로그하지 않는다.
- [x] 세션 원문은 DB에 저장하지 않고, 인증 성공 시 세션을 회전한다.
- [x] 세션 쿠키에 HttpOnly, SameSite=Strict, 제한된 수명을 적용한다.
- [x] 운영 환경에서 Secure 쿠키와 HSTS를 적용한다.
- [x] 모든 상태 변경 요청에 CSRF 토큰을 검증한다.
- [x] 입력값의 길이, 숫자 범위, 열거형 허용 목록을 서버에서 검증한다.
- [x] 사진의 확장자·MIME·시그니처 일치와 전체 PNG/JPEG 디코딩, 파일당 8MB·요청당 32MB·5천만 픽셀 제한을 검사한다.
- [x] 사진을 PNG/JPEG로 재인코딩해 EXIF·후행 데이터를 제거하고 원본 파일명을 저장하지 않는다.
- [x] 채팅 사진은 로그인한 해당 대화 참가자만 조회할 수 있다.
- [x] SQL은 매개변수 바인딩하고 LIKE 메타문자를 이스케이프한다.
- [x] 사용자 콘텐츠는 HTML 문맥 이스케이프하고 CSP를 적용한다.
- [x] 상품 객체 단위의 소유권과 관리자 역할을 서버에서 검사한다.
- [x] 1:1 메시지는 대화 참가자만 조회할 수 있다.
- [x] 잔액 이전과 상품 상태 변경은 한 DB 트랜잭션으로 처리한다.
- [x] 오류 응답에 스택 추적·DB 경로·SQL을 노출하지 않는다.
- [x] 로그인 실패를 계정별·IP 전체로 제한하고 존재하지 않는 계정도 동일 비용으로 검증한다.
- [x] 신고 자동 조치에 서로 다른 신고자가 필요하다.
- [x] 감사 로그에 비밀번호·메시지 본문·IP 원문을 기록하지 않는다.

- 인증·세션·거래·관리자 코드 변경 시 전체 테스트를 재실행한다.
- DB 스키마 변경 시 마이그레이션과 백업 복원 테스트를 추가한다.
- 실제 배포 전 HTTPS, 프록시 IP 신뢰 경계, 파일 권한, 백업 암호화를 점검한다.
- 신고 임계값은 운영 데이터와 오탐 결과를 검토해 조정한다.
